# API Design

Skill-Tango operates entirely on **Convex**, traversing React Server Components and traditional REST endpoints by using direct WebSocketRPC connections.

## Core Schema (`convex/schema.ts`)

- `users`: Stores global profile data and XP.
- `courses`: The overarching JSON object generated by the AI Baseline.
- `chapters`: Relational pointers to group lessons.
- `lessons`: The core content nodes.
- `exercises`: Relational array of quiz objects tied to a `lessonId`.

## The AI Pipeline (`convex/aiPipeline.ts`)

These are serverless Actions (which allow external side-effects like calling OpenAI or ElevenLabs, unlike sterile Queries/Mutations).

1. `assessBaseline(topic, targetLevel)`
   - **Role**: Kickoff. 
   - **Returns**: `AssessmentSession` with introductory chat messages.
   
2. `gradeAssessmentAndGenerateCurriculum(topic, targetLevel, modalities, questions, answers)`
   - **Role**: Heavy lifter.
   - **Returns**: Graded score + wildly deeply nested Course Object (Chapters -> Lessons) via GPT-4o.

3. `generateLessonDirect(lessonTitle, chapterTitle, topic, targetLevel)`
   - **Role**: Just-in-Time text and quiz generator.
   - **Returns**: Markdown text + JSON Quiz array.

## External Sub-systems
- **OpenAI Node Client**: Used for generating `gpt-4o-mini` fast text and `gpt-4o` complex curriculum structures.
- **ElevenLabs FETCH**: REST call to text-to-speech `/v1/text-to-speech/:voiceId`. The resulting blob is passed directly to `ctx.storage.store()` to keep it within the Convex CDN.
